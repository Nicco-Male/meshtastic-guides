# Contributing

Grazie per voler contribuire a **Meshtastic Guides**! Questa guida spiega come aprire issue e pull request in modo efficace.

## Lingua

- Le **issue** e le **PR** possono essere scritte in **italiano** o **inglese**.
- Per la documentazione, mantieni la coerenza linguistica con i file esistenti:
  - `.md` → italiano
  - `.en.md` → inglese

## Aprire una issue

1. Verifica se esiste già una segnalazione simile.
2. Usa uno dei template disponibili.
3. Includi sempre:
   - Versione app/firmware (se rilevante).
   - Modello dispositivo e regione LoRa.
   - Passi per riprodurre il problema.
   - Screenshot o log utili.

## Pull request

- Mantieni le PR **piccole e mirate**.
- Aggiorna `docs/reference/changelog.md` quando modifichi le guide.
- Se aggiungi nuove pagine, inseriscile in `mkdocs.yml`.
- Esegui un controllo ortografico di base e verifica i link interni.

## Stile e formattazione

- Usa titoli chiari (`#`, `##`, `###`) e liste puntate per i passaggi.
- Evita frasi troppo lunghe; preferisci istruzioni passo‑per‑passo.
- Mantieni coerente la terminologia tra IT e EN.

## Dove chiedere aiuto

Se non sei sicuro, apri una discussione sul forum Meshtastic o chiedi chiarimenti nella PR.
